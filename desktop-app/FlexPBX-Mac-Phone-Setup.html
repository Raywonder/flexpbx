📞 FlexPBX Mac Phone Setup

Configure Mac's built-in phone app to test CallCentric integration

🎯 Goal: Set up Mac telephone app as Extension 101 (Dominique) to test calls with FlexPBX Extension 102
1 Check Mac Phone App Availability

First, let's verify which phone applications are available on your Mac:

# Check for phone applications ls /Applications/ | grep -i phone ls /Applications/ | grep -i call ls /Applications/ | grep -i sip # Common Mac phone apps: # - Phone (if iPhone nearby) # - FaceTime (video/audio calls) # - Third-party SIP apps from App Store
⚠️ Note: Mac's built-in Phone app typically requires an iPhone nearby for cellular calling. For SIP calling, we'll need a SIP client app.

4 Configure Advanced Settings

Optimize settings for best call quality and FlexPBX compatibility:

🎵 Audio Settings:

Preferred Codecs (in order): 1. G.722 (HD Audio - 16kHz) 2. G.711u (PCMU - Standard quality) 3. G.711a (PCMA - European standard) 4. G.729 (Compressed - if bandwidth limited) DTMF Method: RFC2833 (out-of-band) Echo Cancellation: Enabled Noise Suppression: Enabled Automatic Gain Control: Enabled

🔧 Network Settings:

NAT Traversal: Enable if behind router STUN Server: stun.callcentric.com:3478 Registration Refresh: 300 seconds Keep-Alive: Enabled Local SIP Port: 5060 (or auto) RTP Port Range: 10000-20000

📞 Call Features:

Call Waiting: Enabled Call Transfer: Enabled (REFER method) Call Hold: Enabled Conference Calls: 3-way minimum Voicemail Support: Enabled Caller ID: Show/Send enabled
5 Test SIP Registration

Verify your Mac SIP client connects to CallCentric:

✅ Success Indicators:
  • Status shows "Registered" or "Online"
  • Green indicator light in SIP app
  • Extension 101 appears available
  • No error messages in logs
❌ Common Issues & Solutions:
  • Registration Failed: Check username/password
  • Network Error: Verify internet connection
  • Auth Failed: Confirm CallCentric credentials
  • Timeout: Check firewall/router settings

🎯 FlexPBX Integration Testing

Now let's test calls between your Mac (Ext 101) and FlexPBX (Ext 102):

6 Test Extension-to-Extension Calls

📞 Test Scenarios:

Test 1: Basic Call 1. From Mac (Ext 101): Dial 102 2. FlexPBX should ring 3. Answer and verify audio quality 4. Test both directions of audio 5. Hang up properly
Test 2: DTMF Tones 1. During call, press digits 1-9, *, 0, # 2. Verify tones are heard on both ends 3. Test rapid digit entry 4. Confirm all tones transmit clearly
Test 3: Call Hold 1. Put call on hold from Mac 2. Verify hold music plays 3. Resume call successfully 4. Test hold from FlexPBX side
Test 4: Call Transfer 1. Transfer call to another extension 2. Test blind transfer (immediate) 3. Test attended transfer (announce first) 4. Verify transfer completes properly

🎵 Audio Quality Checklist:

  • ✅ Clear voice transmission both directions
  • ✅ No echo or feedback
  • ✅ No crackling or distortion
  • ✅ Consistent volume levels
  • ✅ DTMF tones clear and distinct
  • ✅ No significant delay (latency)
